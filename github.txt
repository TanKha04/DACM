2.8 GitHub

	GitHub là một mạng xã hội đặc biệt dành cho lập trình viên, là một hệ thống quản lý dự án, lưu trữ source code, theo dõi và cộng tác trong các dự án phần mềm.

	Ngoài ra, GitHub là một dịch vụ nổi tiếng cung cấp kho lưu trữ mã nguồn Git cho các dự án phần mềm. GitHub có đầy đủ những tính năng của Git, ngoài ra nó còn bổ sung những tính năng về social để các developer tương tác với nhau.

	GitHub có 2 phiên bản: miễn phí và trả phí. Với phiên bản có phí thường được các doanh nghiệp sử dụng để tăng khả năng quản lý team cũng như phân quyền bảo mật dự án. Còn lại thì phần lớn chúng ta đều sử dụng GitHub với tài khoản miễn phí để lưu trữ source code.

	Nếu một comment để mô tả và giải thích một đoạn code. Thì với GitHub, commit message chính là phần mô tả hành động mà bạn thực hiện trên source code.

	GitHub trở thành một yếu tố có sức ảnh hưởng lớn trong cộng đồng nguồn mở. Cùng với LinkedIn, GitHub được coi là một sự thay thế cho CV của bạn. Các nhà tuyển dụng cũng rất hay tham khảo GitHub profile để hiểu về năng lực coding của ứng viên.

	Trong quá trình phát triển đề tài "Website bán đồ ăn", GitHub được sử dụng để lưu trữ toàn bộ mã nguồn của dự án bao gồm các file PHP xử lý logic nghiệp vụ, HTML xây dựng cấu trúc trang, CSS định dạng giao diện, JavaScript xử lý tương tác và file SQL tạo cơ sở dữ liệu. Việc lưu trữ trên cloud giúp đảm bảo an toàn dữ liệu và tránh mất mát code khi máy tính gặp sự cố.

	GitHub giúp quản lý phiên bản code hiệu quả khi mỗi lần có thay đổi về chức năng như thêm module giỏ hàng, thanh toán, chat giữa khách hàng với cửa hàng, chat giữa khách hàng với shipper hay quản lý đơn hàng, nhóm đều thực hiện commit với message mô tả rõ ràng để theo dõi lịch sử phát triển và dễ dàng quay lại phiên bản trước nếu có lỗi xảy ra.

	GitHub hỗ trợ làm việc nhóm hiệu quả khi các thành viên có thể cùng phát triển các module khác nhau như module Customer cho khách hàng đặt món, module Seller cho người bán quản lý cửa hàng, module Shipper cho người giao hàng nhận đơn và module Admin cho quản trị viên quản lý hệ thống trên các nhánh riêng biệt rồi merge lại khi hoàn thành, giúp tránh xung đột code và tăng năng suất làm việc.

	GitHub còn giúp chia sẻ code với giảng viên hướng dẫn để nhận góp ý và đánh giá tiến độ dự án một cách thuận tiện thông qua việc cấp quyền truy cập repository.
