2.9 Postman

	Postman là một công cụ phổ biến dùng để kiểm thử và phát triển API. Postman cho phép lập trình viên gửi các yêu cầu HTTP đến server và nhận phản hồi một cách trực quan, giúp việc kiểm tra API trở nên đơn giản và hiệu quả.

	Postman hỗ trợ đầy đủ các phương thức HTTP như GET, POST, PUT, DELETE, PATCH. Ngoài ra, Postman còn cho phép thiết lập headers, body, parameters và authentication một cách dễ dàng thông qua giao diện đồ họa thân thiện.

	Postman có 2 phiên bản: miễn phí và trả phí. Phiên bản miễn phí đã đủ đáp ứng nhu cầu kiểm thử API cơ bản. Phiên bản trả phí cung cấp thêm các tính năng nâng cao như làm việc nhóm, monitoring và documentation chuyên nghiệp.

	Postman cho phép lưu trữ các request thành collections, giúp tổ chức và quản lý các API endpoint một cách có hệ thống. Điều này rất hữu ích khi dự án có nhiều API cần kiểm thử.

	Trong quá trình phát triển đề tài "Website bán đồ ăn", Postman được sử dụng để kiểm thử các API của hệ thống bao gồm API lấy danh sách sản phẩm, API thêm sản phẩm vào giỏ hàng, API kiểm tra đơn hàng mới và API gửi tin nhắn. Việc sử dụng Postman giúp phát hiện lỗi sớm và đảm bảo API hoạt động đúng trước khi tích hợp vào giao diện.

	Postman giúp kiểm tra response trả về từ server bao gồm status code, headers và body data. Nhờ đó, lập trình viên có thể xác định API đã trả về đúng dữ liệu hay chưa, có lỗi gì xảy ra không và thời gian phản hồi có chấp nhận được không.

	Postman hỗ trợ tạo môi trường (environment) để quản lý các biến như URL server, token xác thực. Điều này giúp dễ dàng chuyển đổi giữa môi trường development và production khi kiểm thử.

	Postman còn có tính năng tự động hóa kiểm thử thông qua Collection Runner, cho phép chạy hàng loạt request và kiểm tra kết quả một cách tự động, tiết kiệm thời gian kiểm thử thủ công.
